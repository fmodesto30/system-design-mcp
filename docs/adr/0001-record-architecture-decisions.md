# ADR-0001 — Registrar decisões de arquitetura (ADRs)

- **Status:** aceito
- **Contexto:** o Lab toma várias decisões (stack, fonte de dados, fronteiras) que
  precisam ser rastreáveis — coerente com a própria mensagem do projeto de que toda
  decisão técnica tem trade-off e fonte.
- **Decisão:** registrar decisões significativas como ADRs curtos em `docs/adr/`,
  numerados, no formato contexto → decisão → consequências.
- **Consequências:** histórico leve e versionado; cada decisão expõe seu trade-off.
  Custo: disciplina de escrever o ADR junto com a mudança.
